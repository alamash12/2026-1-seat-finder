# Seat Check Firebase UI v9

Firebase Authentication과 Cloud Firestore를 사용하는 모바일 좌석 예약 웹사이트입니다.

## v9 핵심 변경

이 버전은 이전 UI 레이아웃 코드를 부분 수정하지 않고, 좌석 선택/하단바/좌석 관리 화면 관련 구조를 다시 작성했습니다.

### 좌석 관리 화면 레이아웃

좌석 관리 화면은 다음 순서로 고정했습니다.

```txt
상단 화면 전환 버튼
예약 정보 영역
원형 타이머
남는 공간
연장 안내문구
연장하기 버튼
외출하기 / 자리 반납 버튼
```

### 화면 전환 버튼

버튼 자체는 항상 같은 색과 형태를 유지합니다.

- 예약 없음: 삼각형만 흐리게 표시
- 예약 있음: 삼각형 진하게 표시
- 관리 화면 열림: 삼각형 방향 반전

### 연장하기

- 남은 시간이 1시간 이하일 때만 활성화됩니다.
- 클릭하면 남은 시간이 다시 6시간으로 설정됩니다.
- 외출 중에는 비활성화됩니다.

### 외출

- 예약 한 번당 한 번만 가능합니다.
- 외출하기 클릭 시 확인 팝업이 나타납니다.
- 외출 중에는 하단바에 `외출중`으로 표시됩니다.
- 외출 복귀 시 기존 예약 남은 시간이 그대로 이어집니다.

### 좌석 배치

정확한 도면 기반은 아니며, 사진에서 확인 가능한 구조를 참고해 만든 근사 배치입니다.
이전 버전에서 겹치던 문제를 줄이기 위해 좌석 그룹 사이 간격을 크게 두었습니다.

### 좌석 배치도 줌 조절 방법

`script.js` 상단에 있는 아래 값을 수정하면 좌석 배치도 줌 느낌을 바꿀 수 있습니다.

```js
const MAP_ZOOM = 0.84;
```

- 숫자를 낮추면 더 줌아웃됩니다. 예: `0.78`
- 숫자를 높이면 더 줌인됩니다. 예: `0.92`
- 너무 낮추면 좌석 그룹이 다시 가까워질 수 있으니 `0.78 ~ 0.95` 사이에서 조절하는 것을 추천합니다.

좌석 하나의 크기는 `style.css` 아래 부분에서 조절할 수 있습니다.

```css
.seat {
  width: 32px;
  height: 32px;
}
```

좌석을 더 작게 보이게 하려면 `30px`, 더 크게 보이게 하려면 `34px`처럼 바꾸면 됩니다.

## UI v12 수정 내용

- 좌석 관리 화면 상단의 여백은 유지하되, 예약 정보 영역은 더 위로 올렸습니다.
- 화면 전환 버튼을 하단바에서 보이는 것과 같은 사다리꼴 탭 형태로 복구했습니다.
- 관리 화면에서도 같은 사다리꼴 버튼을 사용하고, 내부 삼각형만 반대로 뒤집히도록 수정했습니다.
- 버튼 전체의 투명도는 건드리지 않고, 비활성화 시 삼각형만 연하게 보이도록 유지했습니다.
- Firestore 초기화 버전을 `empty-seats-v12`로 올렸습니다.


## UI v13 수정 내용

- 요구사항을 다시 정리하여 화면 전환 버튼 위치를 수정했습니다.
- 좌석 선택 화면의 하단 바 버튼은 사다리꼴이 아니라 이전의 둥근 탭 형태로 되돌렸습니다.
- 사다리꼴 화면 전환 버튼은 좌석 관리 영역 안쪽 상단에만 표시되도록 변경했습니다.
- 관리 화면에서 사다리꼴 버튼 내부의 삼각형은 아래 방향으로 뒤집혀 표시됩니다.
- 버튼 자체는 항상 선명하게 유지하고, 비활성화 여부는 삼각형 투명도만으로 표시합니다.
- Firestore 초기화 버전을 `empty-seats-v13`으로 올렸습니다.


## Fix applied

- 사다리꼴 방향을 위쪽 좁고 아래쪽 넓은 형태로 수정했습니다.
- 버튼 자체는 선명하게 유지하고, 삼각형은 관리 화면에서 아래 방향으로 표시됩니다.


## 화면 전환 버튼 수정

- 좌석 관리 화면의 화면 전환 버튼을 좌석 선택 화면 하단바와 같은 둥근 탭 디자인으로 맞췄습니다.
- 좌석 관리 화면에서는 버튼 안의 삼각형만 아래 방향으로 회전합니다.
- 비활성화 상태에서도 버튼 전체는 흐려지지 않고, 삼각형만 흐려집니다.


## 수정 사항

- 남은 시간 원형 슬라이더가 시간이 줄어들 때 시계방향으로 줄어들도록 수정했습니다.
- 좌석 버튼을 누른 채로 드래그해도 좌석 배치도가 움직이도록 수정했습니다.
- 드래그 후 손을 뗄 때 좌석 클릭이 함께 발생하지 않도록 방지했습니다.


## 추가 수정 사항

- 좌석을 짧게 탭하면 정상적으로 좌석이 선택되도록 수정했습니다.
- 좌석을 누른 채 일정 거리 이상 움직이면 좌석 선택이 아니라 드래그 스크롤로 처리됩니다.
- 좌석 버튼의 `touch-action`을 조정해 모바일 탭 동작이 막히지 않도록 수정했습니다.


## 추가 수정 사항

- 예약 남은 시간이 00:00:00이 되면 좌석이 자동 반납되도록 수정했습니다.
- 자동 반납 시 안내 알림이 표시되고, 좌석 선택 화면으로 돌아갑니다.
- 좌석 선택 화면에서 하단 바를 위로 드래그하면 좌석 관리 화면이 열리도록 수정했습니다.
- 좌석 관리 화면에서 버튼이 아닌 영역을 아래로 드래그하면 좌석 선택 화면으로 돌아가도록 수정했습니다.
- 버튼, 입력창, 링크를 드래그할 때는 화면 전환 드래그가 실행되지 않도록 예외 처리했습니다.
